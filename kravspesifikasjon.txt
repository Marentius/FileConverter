Kravspesifikasjon (høy nivå) 
Mål: Universelt filkonverteringsverktøy med batch-støtte, for “alt fra HEIC→PNG til 
DOCX→PDF, PDF→DOCX, PPTX→PDF, osv.” 
Plattformer: Windows 10+, macOS 12+, Linux (Ubuntu/Debian). 
Lisens: MIT eller Apache-2.0 (du velger). 
Ytelse: Skal håndtere batch på 1000+ filer med kø og parallellisering (konfigurerbare 
“workers”). 
Pålitelighet: Hver fil behandles atomisk (temp-fil → rename). Robust feilhåndtering, 
detaljerte loggfiler. 
Sikkerhet/Personvern: Lokal kjøring, ingen opplasting til tredjepart. Telemetri = av som 
default. 
Utvidbarhet: Plugins/adapters for nye konverterere (f.eks. ny “heic→webp” modul) uten å 
endre kjernen. 
 
Arkitektur 
1. Core (CLI / Library) 
o Språk: TypeScript/Node.js (enkelt å orkestrere eksterne CLI-verktøy), 
eller Python. Anbefaling: Node.js + TypeScript. 
o Ansvar: 
▪ Oppdage filtype (MIME/signatur), mappe til støttet konvertering. 
▪ Starte underliggende motorer (ImageMagick, LibreOffice, Pandoc, 
FFmpeg, Ghostscript, etc.) via child_process. 
▪ Kø, parallellisering, retry-policy. 
▪ Presets (kvalitet/oppløsning/komprimering). 
▪ Loggføring (JSON + menneskelesbar). 
▪ Konfig (YAML/JSON) + miljøvariabler. 
o Distribusjon: npm + prebuild binærer, eller pakket med pkg/nexe til enkel 
exe/bin. 
2. GUI (valgfri) 
o Tauri + React + Vite (lett vekt, native installere). 
o Funksjoner: drag-and-drop, mappe-modus, køvisning, presets, feillogg, 
historikk. 
3. Adapters (motorer) 
o Bilde: ImageMagick eller libvips (sharp), libheif for HEIC. 
o Dokument: LibreOffice (soffice) headless for docx/xlsx/pptx ↔ pdf, 
Pandoc for txt/md/html/rtf ↔ docx/pdf (ofte via LaTeX). 
o PDF: Ghostscript (komprimering/optimalisering), qpdf 
(sammenslåing/deling), ocrmypdf (valgfritt OCR). 
o Video/lyd (om ønsket): FFmpeg. 
o Arkiv (om ønsket): 7-Zip. 
 
Støttematrise (MVP) 
BILDE 
• Inngang: HEIC, JPG, PNG, WEBP, TIFF, BMP, GIF 
• Utgang: PNG, JPG, WEBP, TIFF 
• Motor: magick (ImageMagick) eller sharp (libvips) 
• Parametre: kvalitet, max-bredde/høyde, strip metadata, fargerom. 
DOKUMENT 
• DOCX → PDF (LibreOffice headless / soffice) 
• PPTX → PDF (LibreOffice) 
• XLSX → PDF (LibreOffice) 
• PDF → DOCX (LibreOffice; tap av layout kan forekomme) 
• MD/HTML → PDF/DOCX (Pandoc) 
• RTF/TXT ↔ DOCX/PDF (Pandoc/LibreOffice) 
PDF-OPERASJONER 
• Komprimer PDF (Ghostscript) 
• Slå sammen / splitte (qpdf) 
• OCR til søkbar PDF (ocrmypdf, valgfritt) 
VIDEO/LYD (valgfri senere) 
• MP4/MOV → MP4/Web-friendly (FFmpeg presets) 
• Ekstraher lyd (FFmpeg) 
Ikke-funksjonelle krav 
• Ytelse: Parallellisering (default: workers = cpu_cores - 1), kø med back-pressure. 
• Minnebruk: Strømme-baserte pipes der mulig (unngå å laste hele filer i RAM). 
• Robusthet: Maks n retries (default 2). Partial failure rapporteres per fil. 
• Logging: 
o Console: human-readable. 
o Fil: JSON-linjer (nivå: info/warn/error) + separate motor-stderr. 
• Internasjonalisering: UI: engelsk + norsk (JSON i18n-filer). 
• Tilgjengelighet: Tastaturnavigasjon, status/aria-labels i GUI. 
• Installering: 
o Windows: installer m/ “check dependencies” og knapp “Installer 
LibreOffice/ImageMagick”. 
o macOS: notarized dmg (hvis mulig), brew install-instrukser for 
avhengigheter. 
o Linux: apt/yum/pacman oppskrifter.
